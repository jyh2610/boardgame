🗡️ 레지스탕스 아발론 (The Resistance: Avalon) 웹 게임 코어 엔진 기획서1. 게임 개요 및 핵심 루프장르: 소셜 디덕션 (정체 숨기기/마피아), 투표, 팀 빌딩핵심 목표:선의 세력 (아서 왕의 부하): 5번의 퀘스트 중 3번을 성공시킨 후, 암살자로부터 멀린을 지켜내기.악의 세력 (모드레드의 하수인): 5번의 퀘스트 중 3번을 실패시키거나, 원정대 구성 투표를 5번 연속 부결시키거나, 마지막에 진짜 멀린을 암살하기.게임 흐름: 라운드 시작 ➔ 원정대장 지목 (팀 구성) ➔ 전원 찬반 투표 ➔ (가결 시) 원정대원 퀘스트 카드 제출 ➔ 결과 공개 ➔ 다음 라운드.2. 게임 상태 (State) 모델링 요구사항서버(엔진)가 절대적으로 비밀을 통제하며 클라이언트에 제한된 정보만 내려주어야 합니다.Match State (전체 게임 상태):Current Round: 1 ~ 5Quest Track: 길이 5의 배열 (각 라운드의 성공/실패 여부 기록)Reject Track: 0 ~ 5 (투표 부결 횟수. 원정이 출발하면 0으로 초기화. 5 도달 시 악 승리)Phase: NIGHT ➔ TEAM_BUILDING ➔ VOTING ➔ QUESTING ➔ ASSASSINATION ➔ ENDPlayer State:Role: 멀린, 퍼시벌, 일반 선 / 암살자, 모르가나, 모드레드, 오베론, 일반 악Team: GOOD / EVILIs Leader: 현재 원정대장 여부 (true/false)Vote Token: 이번 턴에 낸 투표 (Approve / Reject) - 투표 종료 전까지 비공개Quest Card: 원정대원이 낸 퀘스트 결과 (Success / Fail) - 제출 즉시 섞여서 비공개 처리3. 초기화 (Setup) 및 밤 단계 (Night Phase) 로직3.1. 인원별 역할 및 원정대 규모 세팅엔진은 시작 인원에 따라 역할 풀을 구성하고 무작위 분배합니다.총 인원선 : 악 비율퀘스트별 필요 원정대원 수 (1~5라운드)비고5명선 3 : 악 22 - 3 - 2 - 3 - 36명선 4 : 악 22 - 3 - 4 - 3 - 47명선 4 : 악 32 - 3 - 3 - 4* - 4*4R는 실패 2장 필요8~9명선 5/6 : 악 33 - 4 - 4 - 5* - 5*4R는 실패 2장 필요10명선 6 : 악 43 - 4 - 4 - 5* - 5*4R는 실패 2장 필요3.2. 정보 비대칭 매트릭스 (Night Phase Vision)게임 시작 시, 엔진은 각 플레이어의 클라이언트에게 본인이 볼 수 있는 정보만 JSON 페이로드로 내려주어야 합니다. (매우 중요)멀린: 모든 악(EVIL) 플레이어를 알 수 있음. 단, 모드레드는 선으로 보임. (누가 어떤 악 직업인지는 모름, 그냥 '악'으로만 보임)퍼시벌: 멀린과 모르가나를 알 수 있음. (둘 다 '멀린'으로 보이며 구분 불가)악의 세력 (일반 악, 암살자, 모르가나, 모드레드): 서로가 누군지 알 수 있음. 단, 오베론은 악에게 보이지 않음.일반 선, 오베론: 자신 외에 아무런 정보도 받지 못함.4. 턴 액션 시스템 (Action Resolution)Phase 1: 원정대 구성 (Team Building)액션: 원정대장(Is Leader == true)은 현재 라운드에 필요한 인원수만큼 플레이어를 선택하여 API로 전송. (자신 포함 가능)검증: 요구 인원수와 일치하지 않으면 엔진에서 Reject.Phase 2: 찬반 투표 (Voting)액션: 모든 플레이어가 해당 원정대에 대해 찬성(Approve) 또는 반대(Reject)를 제출.해결 로직 (Resolution):전원이 투표할 때까지 상태 대기.투표가 완료되면 동시에 모두 공개.찬성이 과반수(> 50%)일 경우: 가결 ➔ 원정대 출발 (Phase 3로 이동), Reject Track = 0.찬성이 절반 이하(<= 50%)일 경우: 부결 ➔ 원정대장을 다음 사람으로 넘김, Reject Track + 1.부결 트리거: Reject Track이 5가 되는 순간 게임 즉시 종료 ➔ 악의 승리.Phase 3: 퀘스트 수행 (Questing)액션: 가결된 원정대에 속한 플레이어들만 퀘스트 카드(성공/실패)를 제출.검증 (강제 룰):선(GOOD) 진영 플레이어는 무조건 **'성공'**만 제출 가능 (엔진단에서 '실패' 제출을 막아야 함).악(EVIL) 진영 플레이어는 '성공', '실패' 중 자유롭게 선택 가능.해결 로직 (Resolution):카드가 모두 모이면 누가 무엇을 냈는지 알 수 없도록 순서를 섞어서(Shuffle) 공개.'실패' 카드가 1장이라도 나오면 해당 라운드 원정 실패.(예외 룰) 7인 이상 게임의 4라운드는 '실패' 카드가 2장 이상 나와야 실패 처리.5. 게임 종료 및 암살 (Endgame State)악의 즉시 승리 조건: 퀘스트가 3번 실패하거나, 투표가 5연속 부결될 경우.선의 조건부 승리: 퀘스트가 3번 성공할 경우 ➔ Phase 4: 암살 단계(Assassination) 진입.Phase 4: 암살 단계 (Assassination)액션: 암살자 역할을 가진 플레이어가 멀린이라고 생각되는 플레이어 1명을 지목. (악 세력끼리 채팅/음성으로 토론 후 암살자가 최종 결정)결과 판정:지목된 타겟이 멀린일 경우 ➔ 악의 역전 승리.지목된 타겟이 멀린이 아닐 경우 ➔ 선의 최종 승리.💡 엔진 개발을 위한 추가 팁 (선택 규칙 상태 관리)만약 프로모션(호수의 여신, 엑스칼리버, 랜슬롯)을 추가하실 계획이라면 엔진 설계 시 아래 훅(Hook)을 미리 뚫어두는 것이 좋습니다.호수의 여신: 투표/퀘스트 종료 후 발생하는 추가 Phase (1명 타겟팅 ➔ 대상의 선/악 상태값을 여신 소유자에게만 1:1로 반환).엑스칼리버: Phase 3(퀘스트 수행) 직후, 특정 1인의 퀘스트 카드를 강제로 뒤집는 로직.이 기획서를 안티그래비티 쪽에 전달하시면 상태 관리 뼈대를 잡는 데 큰 무리가 없을 것입니다. 아발론은 투표 동시 공개와 익명 셔플 로직이 핵심인데, 이 부분에 대한 소켓(WebSocket) 통신 이벤트 흐름(Sequence)도 제가 정리해 드릴까요?
